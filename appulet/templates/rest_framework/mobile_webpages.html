Mobile Webpages

In addition to the API endpoints, the following URLs may be used to access webpages formatted for mobile devices.

Manual

Used in webview accessed from app's main switchboard.

Usage

http://holet.movelab.net/<language code>/manual/_mob/

Parameters

  • language code: The code for the user's current language preference. (Aranese: oc, Spanish: es, Catalan: ca, French: fr, English: en.)

Examples

  • http://holet.movelab.net/en/manual/_mob/

User Registration

Used in webview accessed from app's menu. If the user successfully registers for a new account, the server response includes a div element with id=credentials, containing a JSON string as its inner html. The JSON string has keys for username and token, which should be saved by the device and used for subsequent REST requests.

Usage

http://holet.movelab.net/<language code>/register/mob/

Parameters

  • language code: The code for the user's current language preference. (Aranese: oc, Spanish: es, Catalan: ca, French: fr, English: en.)

Examples

  • http://holet.movelab.net/en/register/mob/

Survey

Used in webview that pops up when user exists from the detailed itinerary view.

Usage

http://holet.movelab.net/<language code>/survey/mob/route_survey/<route id>/

Parameters

  • language code: The code for the user's current language preference. (Aranese: oc, Spanish: es, Catalan: ca, French: fr, English: en.)
  • route id: The server id (integer) of the route that the user was following, and about which the user's survey responses will be focused.

Examples

  • http://holet.movelab.net/en/survey/mob/route_survey/10/
